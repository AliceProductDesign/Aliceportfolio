The Challenge

TiriVelo's pet care marketplace was losing users at the most critical moment: provider evaluation.

Search context disappeared mid-browse, filters looked functional but weren't. Provider profiles lacked the trust signals users needed to commit, and a silent edge case allowed wrong-service bookings with no warning.

Users were reaching the decision point and losing confidence — not because providers were bad, but because the product wasn't giving them what they needed to decide.

The Solution

End-to-end redesign of the Search, Discovery, and Provider Profile flows for a real product in active development.

Persistent search context banners, a rebuilt filter system with grouped mental models, trust-signal-rich provider profiles, a three-tier CTA hierarchy, and an inline solution to the booking edge case — validated through moderated usability testing.

The Problem

Before a pet owner books a stranger, they need to feel certain. Not mostly sure.

TiriVelo is a pet care marketplace helping owners discover and book local pet care providers. When I joined the project, the booking experience was breaking down at exactly that critical moment.

Users were reaching the provider evaluation stage and losing confidence, not because the providers weren't good, but because the product wasn't giving them what they needed to decide.

Specific problems: users lost search context after entering results; filtering was visually present but functionally broken; provider cards had weak trust signals; key information was fragmented behind tabs.

Diagnostic findings from the audit

Important info like pricing, reviews, and availability was hidden behind disconnected tabs and extra taps — no review count/summary, no pricing visible on the profile landing page, and background check badge absent.

The filter button existed visually but opened nothing, preventing users from narrowing results.

The search flow asked for too much information before users were ready to commit.

All CTAs used the same visual style, reducing action hierarchy — creating a lack of visual distinction between high-risk and low-risk actions.

Missing home environment details reduced provider transparency, and users had to scroll excessively just to view provider availability.

Provider cards lacked the key information needed for confident decision-making at a glance.

Redesigning Search

Orientation is a trust signal. When users feel lost, they feel uncertain.

The redesign introduced:

  • A persistent search context banner
  • Provider status indicators ("Active Now," "Trusted by TiriVelo")
  • Visible sort-state labels

The underlying principle: if users know where they are, they can decide with confidence.

Provider Profiles

Profiles are where booking decisions are made or abandoned.

Key additions included:

  • Photo-first trust building
  • Verification badges
  • Repeat-client metrics
  • Environment information
  • Persistent booking actions
  • Availability reassurance
  • Review credibility indicators
[Provider profile — verification badges, ratings, availability calendar]

The dynamic availability banner was designed to reduce booking uncertainty by adapting to the user's search context. If no date was selected, it encouraged exploration with "Check Availability"; date-specific searches provided immediate availability feedback to help users evaluate providers faster and with greater confidence.

Solving a Critical Edge Case

Wrong service selected silently → modal fatigue → inline solution.

During our first round of user testing, users could accidentally book a different service than the one they originally searched for.

A confirmation modal was initially attempted to solve the issue but created modal fatigue.

The final solution used inline selection states, persistent original search context, and clear service-switch messaging.

The Outcome

A real product, with redesigned flows forming the live product foundation moving towards launch.

To support scalability and consistency across the marketplace experience, the redesign expanded TiriVelo's design system and interaction framework with:

Combined service-category shapes with the magnifier to create a more specific and recognizable "browse services" interaction.

Renamed "Message" to "Inbox" to create clearer expectations around provider communication and booking updates.

Replaced the underline indicator with a filled background state to create a stronger, more scannable active-tab affordance.

Standardized icon styling and visual weight for consistency across navigation. Created filled and outline icon variants for clearer active/inactive states, improved accessibility, and native mobile interaction patterns.

Replaced the single-button system with a three-tier CTA hierarchy that visually distinguishes high-commitment booking actions from secondary and low-risk utility actions.

Component organization built to hand off cleanly into a React Native codebase.

Ensured compliance with Apple HIG and WCAG touch target standards.

Reflections

What the Testing Taught Me

The original services tab was designed to be non-interactive, display only. Every single participant attempted to tap service rows. The final iteration made them interactive, redesigning around what users naturally expected rather than what the design intended. User expectations consistently outweigh designer assumptions.

What I'd Do Differently

With more time, I would have run a second round of testing specifically on the edge case solution, and pushed further into the provider onboarding experience to understand trust from both sides of the marketplace.
